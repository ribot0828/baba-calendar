# 🐎 馬場カレンダー

競馬の各開催場の**馬場情報・血統傾向**を日付ベースで振り返るための、単一HTMLアプリ。
サーバ不要・データは端末内のみ・外部送信なし。

## 使い方

`index.html` をブラウザで開くだけ（PC・スマホ両対応）。ダブルクリック、またはスマホのブラウザで開く。

### 画面
- **カレンダー**: 日付をタップ → その日の開催場サマリー → 開催場をタップ → 芝/ダート別の詳細メモ。
  詳細画面では **前年同開催との比較**（回・日目が一致する前年データ、無ければ前年同時期±10日）を表示。
- **履歴**: 開催場別 / 馬場別 / 並び順（開催場順・日付順）でメモ履歴を一覧。
- **データ**: CSV / JSON の一括入出力。

### メモの追加
右下の「＋」→ ①開催場を選ぶ → ②馬場（芝/ダート/障害）を選ぶ → 馬場状態・回/日目・血統タグ・本文を入力して保存。
※ **回・日目**を入れておくと前年比較の精度が上がります。

## インストール（PWA・アプリ化）

GitHub Pages等のURLで開くと、**アプリとしてインストール**できます（審査不要・無料）。

- **iPhone(Safari)**: 共有 → 「ホーム画面に追加」。
- **Android(Chrome)**: メニュー → 「アプリをインストール」/「ホーム画面に追加」。
- **PC(Chrome/Edge)**: アドレスバー右のインストールアイコン。

インストール後はアイコン起動・全画面表示・**オフラインでも起動**（Service Workerがアプリ本体をキャッシュ）。ページ本体はオンライン時にネットワーク優先で取得するため、更新も自動で反映されます。
※ 構成ファイル: `manifest.webmanifest`, `sw.js`, `icon-192/512/180.png`。アプリ更新時は `sw.js` の `CACHE` 名（`baba-cal-v1`→`v2`…）を上げてください。

## JRA開催日程（内蔵）

2025・2026年のJRA全開催日を**公式日程PDF（`program/<年>/pdf/nittei.pdf`）から抽出して内蔵**しています。

- カレンダーに開催場が自動表示（例: 6/1 → 東京・京都）。
- 日付→開催場で「○回○日目」を表示（例: 2回東京12日目）。
- メモ追加時、JRA開催日なら**回・日目が自動入力**される。
- 内蔵日程はユーザーのメモとは別管理。**履歴・CSV/JSONには含まれない**（読み取り専用レイヤー）。

**データ検証**: 抽出結果は公式の年間集計（各競馬場の開催回数・日数、計36回288日）と**両年とも完全一致**。主要G1（日本ダービー=2回東京12日目、有馬記念=5回中山8日目 等）も照合済み。2026年は中京4回24日・京都5回46日など2025年と異なる点も正しく反映。

## 重賞スケジュール（内蔵）

過去レースと馬場メモを紐づけられるよう、**重賞（GⅠ〜GⅢ・障害）を内蔵**しています。

- 日付をタップ → その日の開催場ごとに「重賞名＋グレード」を表示（例: 東京 → 日本ダービー(GⅠ)・目黒記念(GⅡ)）。
- カレンダーの日セルに **GⅠ開催日は赤い「GⅠ」バッジ**を表示（大きなレース日がひと目で分かる）。
- グレードは色分け（GⅠ=赤 / GⅡ=橙 / GⅢ=青 / 障害J・G=紫）。
- 開催日程と同じく読み取り専用レイヤー（メモ・履歴・CSVには非含有）。

**出典・検証**: JRA公式映像の「レースカレンダー」(prc.jp) のHTMLを直接パースして抽出（Web要約は使わず構造から取得）。2025年=140レース（通年）、2026年=73レース（上半期の実施済み分。7月以降は順次）。全213件のうち211件が内蔵の開催日程（開催場・開催日）と一致。残り2件（2026年2月の東京新聞杯・きさらぎ賞）は悪天候順延で火曜開催になった実績日で、公式結果通り。

## データ入出力（可逆）

- **CSV**（推奨・Excel等で編集可）: 列は
  `id,date,venue,round,day,surface,condition,tags,memo,createdAt,updatedAt`
  - `tags` は `;`（セミコロン）区切り。
  - インポート時、`id` が既存と一致する行は **上書き更新**、無ければ新規追加（重複しない＝可逆）。
  - 「CSVテンプレDL」でサンプル付き雛形を取得可能。
- **JSON**: 完全バックアップ用。
- **範囲を指定して保存**: 全件エクスポートに加え、**年・月・開催場・回・馬場**で絞り込んだ分だけCSV/JSONで書き出せる（該当件数を表示）。ファイル名に条件が入る（例: `baba_2025_6月_東京_芝.csv`）。JSONには適用したフィルタ条件も記録。

日付は `YYYY-MM-DD` 形式。

## セキュリティ / 配布

コミュニティ配布を想定した設計：

- **サーバなし**の単一HTML → バックエンド侵害リスクなし。GitHub Pages等に静的ファイルとして置くだけで配布可能。
- **外部通信を全面遮断**（CSP `connect-src 'none'`）→ メモが外部に送信されない。トラッキングなし。
- **外部CDN不使用** → サプライチェーン汚染の余地なし。
- **XSS対策**: メモ・タグは全てテキストとして描画（`innerHTML` に利用者入力を入れない）。
- **CSVインジェクション対策**: `= + - @` で始まるセルは出力時に `'` を付与し、取込時に除去（表計算ソフトでの数式暴発を防止しつつ可逆）。
- 保存先は端末のブラウザ内（localStorage）。**個人情報を保存しない運用**を推奨。

### 配布時の注意
- localStorage はドメイン（オリジン）ごとに分離されます。配布先URLが変わると各利用者のデータは引き継がれません（各自でCSV/JSONエクスポートしてバックアップ）。
- 改変版を再配布する場合も、CSPと「テキスト描画」の原則を崩さないでください。
