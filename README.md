# 🐎 馬場カレンダー

競馬の各開催場の**馬場情報・血統傾向**を日付ベースで振り返るための、単一HTMLアプリ。
サーバ不要・データは端末内のみ・外部送信なし。

> **⚠️ 非公式のファンツールです。** JRA（日本中央競馬会）・関連団体とは一切関係なく、公認・提携もされていません。詳しくは[免責事項](#免責事項非公式について)をご覧ください。

## 使い方

`index.html` をブラウザで開くだけ（PC・スマホ両対応）。ダブルクリック、またはスマホのブラウザで開く。

### 画面
- **カレンダー**: 日付をタップ → その日の開催場サマリー → 開催場をタップ → 芝/ダート別の詳細メモ。
  詳細画面では **前年同開催との比較**（回・日目が一致する前年データ、無ければ前年同時期±10日）を表示。
- **履歴**: 開催場別 / 馬場別 / 並び順（開催場順・日付順）でメモ履歴を一覧。
- **データ**: CSV / JSON の一括入出力。

### メモの追加
右下の「＋」→ ①開催場を選ぶ → ②馬場（芝/ダート/障害）を選ぶ → 馬場状態・回/日目・血統タグ・本文を入力して保存。
※ **回・日目**を入れておくと前年比較の精度が上がります。

## インストール（PWA・アプリ化）

GitHub Pages等のURLで開くと、**アプリとしてインストール**できます（審査不要・無料）。

- **iPhone(Safari)**: 共有 → 「ホーム画面に追加」。
- **Android(Chrome)**: メニュー → 「アプリをインストール」/「ホーム画面に追加」。
- **PC(Chrome/Edge)**: アドレスバー右のインストールアイコン。

インストール後はアイコン起動・全画面表示・**オフラインでも起動**（Service Workerがアプリ本体をキャッシュ）。ページ本体はオンライン時にネットワーク優先で取得するため、更新も自動で反映されます。
※ 構成ファイル: `manifest.webmanifest`, `sw.js`, `icon-192/512/180.png`。アプリ更新時は `sw.js` の `CACHE` 名（`baba-cal-v1`→`v2`…）を上げてください。

## JRA開催日程（内蔵）

2023〜2026年のJRA全開催日を内蔵しています（2025・2026は公式日程PDF `program/<年>/pdf/nittei.pdf`、2023・2024はJRA公式カレンダー prc.jp のグリッドから抽出）。

- カレンダーに開催場が自動表示（例: 6/1 → 東京・京都）。
- 日付→開催場で「○回○日目」を表示（例: 2回東京12日目）。
- メモ追加時、JRA開催日なら**回・日目が自動入力**される。
- 内蔵日程はユーザーのメモとは別管理。**履歴・CSV/JSONには含まれない**（読み取り専用レイヤー）。

**データ検証**: 抽出結果は公式の年間集計（各競馬場の開催回数・日数、計36回288日）と**両年とも完全一致**。主要G1（日本ダービー=2回東京12日目、有馬記念=5回中山8日目 等）も照合済み。2026年は中京4回24日・京都5回46日など2025年と異なる点も正しく反映。

## 重賞スケジュール（内蔵）

過去レースと馬場メモを紐づけられるよう、**重賞（GⅠ〜GⅢ・障害）を内蔵**しています。

- 日付をタップ → その日の開催場ごとに「重賞名＋グレード」を表示（例: 東京 → 日本ダービー(GⅠ)・目黒記念(GⅡ)）。
- カレンダーの日セルに **GⅠ開催日は赤い「GⅠ」バッジ**を表示（大きなレース日がひと目で分かる）。
- グレードは色分け（GⅠ=赤 / GⅡ=橙 / GⅢ=青 / 障害J・G=紫）。
- 開催日程と同じく読み取り専用レイヤー（メモ・履歴・CSVには非含有）。

**出典・検証**: JRA公式映像の「レースカレンダー」(prc.jp) のHTMLを直接パースして抽出（Web要約は使わず構造から取得）。2023年=139、2024年=139、2025年=140（各通年）、2026年=73（上半期の実施済み分。7月以降は順次）。開催日程（開催場・開催日）と相互整合を確認済み。2023年は京都改修で阪神開催が多く、2024年は阪神改修で宝塚記念等が京都開催──といった会場移設も実績通り正確に反映。

## 一般メモ（開催日に依存しないメモ）

下部タブの「🗒️ 一般メモ」。特定の開催日に紐づかないメモを**項目ごと**に蓄積できます。

- **項目の種類**：
  - **JRA開催場**（10場から選択）
  - **馬名**（日本語名。英名も任意で登録可。**大文字小文字を問わず同一馬として自動でまとまる**／英名優先で判別）
  - **自由**（海外・地方など自由入力。例: 香港シャティン、大井、凱旋門賞）
- 項目を選ぶ／作ると、そこに**メモを自由に追加**（1項目に複数メモ可）。各メモにタグも付けられる。
- **項目別に一覧表示**。検索窓で項目名・英名・メモ本文を横断検索（大小文字問わず）。
- **CSV / JSON で保存・読み込み**（一般メモ専用。列: `kind,subject,subjectEn,tags,memo`＋任意で`id`）。データ欄と同じUIで、**エクスポート（全件＋種類/項目で絞り込み）** と **インポート（CSVテンプレDL・🤖AIでCSV作成つき）** を分けて用意。

## バックアップ / 復元（履歴タブ）

データ消失に備え、**カレンダーメモと一般メモをまとめて1つのJSONファイル**に保存できます（履歴タブ下部）。

- **全データを保存**：すべてを1ファイルにバックアップ。
- **選択日より前を保存**：日付を選ぶと、その日より前のカレンダーメモ（開催日基準）と一般メモ（作成日基準）だけを保存（対象件数を表示）。
- **復元**：バックアップJSONを読み込むと現在のデータに統合（同じ`id`は上書き、無ければ追加）。定期的に保存しておけば端末変更や消去時も安心。

## ダート砂の産地割合（内蔵・推移つき）

JRAが2025年から公表し始めた「ダートコースのクッション砂の産地割合」を内蔵。JRAは最新値しか残さないため、**Internet Archive（Wayback Machine）から過去の公表値を復元**し、変遷を辿れるようにしています。

- 開催場の詳細シートに、**その日時点の砂配合**（産地別の積み上げバー＋白砂率）を表示。
- 「産地割合の推移を見る」で、公表開始（2025-03）以降の**変化点**を時系列表示。
- 出典: JRA公式 [ダートコースのクッション砂について](https://www.jra.go.jp/keiba/baba/dirt/) を Wayback で復元（7時点：2025-03/05/08/09, 2026-02/06, 現在）。全時点・全10場で合計100%を検証済み。
- 例: 中山は2025春「青森100%」→秋に豪アルバニー10%導入。福島は2026年に豪アルバニー15%追加。
- 参照JSON: `jra-dirt-sand-history.json`。日付は公表確認日（実際の切替はその前後）。

## データ入出力（可逆）

- **CSV**（推奨・Excel等で編集可）: 列は
  `id,date,venue,round,day,surface,condition,tags,memo,createdAt,updatedAt`
  - `tags` は `;`（セミコロン）区切り。
  - インポート時、`id` が既存と一致する行は **上書き更新**、無ければ新規追加（重複しない＝可逆）。
  - **新規追加なら `id` 列は不要**。必要な列は `date,venue` だけで、他は任意（`surface,condition,round,day,tags,memo`）。
  - 「CSVテンプレDL」で全開催場の記入例つき簡易テンプレ（id・日時列なし）を取得可能。
  - **「🤖 AIでCSV作成」**：手入力が大変なとき用。プロンプトをコピーしてChatGPT/Claude等に自分のメモと一緒に貼ると、取り込めるCSVを生成してくれる。
- **JSON**: 完全バックアップ用。
- **範囲を指定して保存**: 全件エクスポートに加え、**年・月・開催場・回・馬場**で絞り込んだ分だけCSV/JSONで書き出せる（該当件数を表示）。ファイル名に条件が入る（例: `baba_2025_6月_東京_芝.csv`）。JSONには適用したフィルタ条件も記録。

日付は `YYYY-MM-DD` 形式。

## セキュリティ / 配布

コミュニティ配布を想定した設計：

- **サーバなし**の単一HTML → バックエンド侵害リスクなし。GitHub Pages等に静的ファイルとして置くだけで配布可能。
- **外部通信を全面遮断**（CSP `connect-src 'none'`）→ メモが外部に送信されない。トラッキングなし。
- **外部CDN不使用** → サプライチェーン汚染の余地なし。
- **XSS対策**: メモ・タグは全てテキストとして描画（`innerHTML` に利用者入力を入れない）。
- **CSVインジェクション対策**: `= + - @` で始まるセルは出力時に `'` を付与し、取込時に除去（表計算ソフトでの数式暴発を防止しつつ可逆）。
- 保存先は端末のブラウザ内（localStorage）。**個人情報を保存しない運用**を推奨。

### 配布時の注意
- localStorage はドメイン（オリジン）ごとに分離されます。配布先URLが変わると各利用者のデータは引き継がれません（各自でCSV/JSONエクスポートしてバックアップ）。
- 改変版を再配布する場合も、CSPと「テキスト描画」の原則を崩さないでください。
